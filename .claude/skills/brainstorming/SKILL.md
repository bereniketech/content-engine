# Brainstorming Ideas

Help the user fully explore and articulate an idea through natural conversation, then capture everything in a document.

**Rule:** Do NOT propose approaches, make design decisions, suggest implementations, invoke any planning skill, or take any action beyond asking questions and writing the idea doc.

---

## 1. Process

1. **Understand the seed idea** — ask one open-ended question to get the user talking
2. **Ask follow-up questions** — one at a time, dig deeper into any angle that needs more exploration
3. **Reflect back** — periodically summarize what you've heard to confirm you're capturing it right
4. **Write the idea doc** — when the user feels the idea is fully flushed out, write everything to a file

---

## 2. Question Approach

- One question per message — never stack questions
- Prefer open-ended over multiple choice (you're exploring, not converging)
- Follow threads: if the user says something interesting, go deeper on that before moving on
- Cover: what it is, why it matters, who it's for, what makes it different, what excites the user about it
- Stop when the user says they're done, or when the idea feels fully articulated

---

## 3. Writing the Doc

When the user signals they're done (or asks you to write it up):

- Write to `docs/ideas/YYYY-MM-DD-<topic>.md`
- Capture everything from the conversation: the core idea, the angles explored, the nuances, the open questions
- Do NOT reframe, editorialize, or add your own structure beyond what was discussed
- Ask the user if the doc captures everything before closing

**Rule:** After writing the doc, your job is done. Do not suggest next steps, planning, or implementation. If the user wants to go further, they will say so.
