## Objective
Security review of the R6 image generation pipeline changes: verify OWASP compliance, input sanitization, and that image URLs are never exposed to unauthorized users.

## Files

### Review
- `D:/content-engine/app/api/images/route.ts`
- `D:/content-engine/app/api/images/generate/route.ts`
- `D:/content-engine/lib/gemini-image.ts`
- `D:/content-engine/lib/fal-images.ts`

## Security Checklist

1. **Authentication:** All API routes require JWT auth via `requireAuth`. Verify no route is accessible without auth.

2. **Input Sanitization:** 
   - `topic` and `blog` inputs pass through `sanitizeInput()` / `sanitizeUnknown()`
   - `autoGenerate` flag is boolean — no injection surface
   - Image prompts are generated server-side from sanitized inputs, not from user-provided prompts directly

3. **API Key Exposure:**
   - `GEMINI_API_KEY` — never returned in response body
   - `FAL_API_KEY` — never returned in response body
   - Verify both are server-side env vars only (no `NEXT_PUBLIC_` prefix)

4. **Image URL Safety:**
   - Base64 data URLs stored in content_assets — scoped by RLS to session owner
   - fal.ai CDN URLs — public but non-guessable
   - Generated images never exposed to other users (RLS on content_assets)

5. **Error Response Safety:**
   - 500 errors return generic message, not raw exception
   - OWASP comment present in route files

6. **Rate Limiting:**
   - Existing middleware rate limiting applies
   - Verify no bypass possible via the autoGenerate path

7. **Prompt Injection:**
   - Image prompts generated by Claude from sanitized article content — low risk
   - Verify `sanitizeInput()` strips any HTML/script injection from user topic

## Implementation Steps

1. Read each file listed above
2. Verify each item in the security checklist
3. Report findings as inline comments or a security findings list
4. If issues found: create a `security-findings-r6.md` in `.spec/competitive-gaps-roadmap/` with:
   - Severity (High/Medium/Low)
   - Description
   - Remediation

## Acceptance Criteria
- All 7 checklist items verified as compliant.
- No High severity findings unfixed.
- Any Medium findings documented with remediation plan.
- OWASP comment confirmed present in all route files.
