## Objective
Create `BriefCard` component that appears in the output panel after the research step completes and before generation. It displays the structured brief in an editable form. "Generate Article" is disabled until the brief is saved/approved.

## Files

### Create
- `D:/content-engine/components/sections/BriefCard.tsx`

## Dependencies
- TASK-025: `/api/brief` routes exist
- Existing `components/ui/button.tsx`, `components/ui/card.tsx`, `components/ui/badge.tsx`
- `app/dashboard/research/` for integration point

## Codebase Context

The output panel in the existing dashboard renders sections in a tab or step sequence. `ResearchPanel.tsx` is shown after research. The `BriefCard` should appear between `ResearchPanel` and the article generation button.

The brief is generated by calling `POST /api/brief` with the `sessionId` after research completes. This call can be triggered from `BriefCard` on mount if the brief doesn't exist yet.

## Implementation Steps

1. Create `components/sections/BriefCard.tsx`:

```typescript
'use client'
interface BriefCardProps {
  sessionId: string
  onBriefApproved: () => void    // called when status='approved', enables Generate button
}
```

2. On mount:
   - Fetch `GET /api/brief?sessionId={sessionId}`
   - If 404: call `POST /api/brief` with `{ sessionId }` (auto-generate from research)
   - If success: populate form fields from response

3. Form fields:
   - **Keyword**: text input
   - **Search Intent**: select (informational / commercial / transactional)
   - **Audience**: text input
   - **Suggested H1**: text input
   - **H2 Outline**: sortable list where each item is an editable text input; "Add Section" button appends blank item; drag handle (emoji "⠿") for reordering (simple index swap on drag end)
   - **Recommended Word Count**: number input
   - **CTAs**: comma-separated tag input

4. Buttons:
   - **"Save Brief"**: calls `PATCH /api/brief/{id}` with current field values + `{ status: 'approved' }` → on success calls `onBriefApproved()`
   - **"Regenerate"**: calls `POST /api/brief` again → refreshes form
   - "Save Brief" shows spinner while loading

5. Loading state: skeleton card while initial fetch is in progress.

6. Error state: "Failed to generate brief — [Retry] button" if POST fails.

## Test Cases

- Mount with no existing brief → auto-generates → form shows populated fields
- Mount with existing brief → form pre-filled with existing data
- Edit keyword field → click Save Brief → PATCH called with new keyword + status=approved
- onBriefApproved called after successful save
- Regenerate button → new POST → form refreshes
- Loading skeleton shown during initial fetch

## Decision Rules
- Auto-generate on mount (no user click required) — reduces friction.
- H2 Outline reorder: simple swap on drag, not full drag-drop (no libraries).
- "Save Brief" always sets `status: 'approved'` — user editing and saving = approval.
- `onBriefApproved` is the only way the parent enables the Generate button.

## Acceptance Criteria
- BriefCard auto-generates brief on first mount if none exists.
- All fields editable in place.
- "Save Brief" calls PATCH and triggers `onBriefApproved`.
- Loading skeleton shown during fetch/generate.
- Error state with retry button shown on failure.
